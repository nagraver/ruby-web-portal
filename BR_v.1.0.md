# Документация проекта Portals

## 1. Бизнес-требования (Business Requirements)

### 1.1 Цель проекта
Создать информационно-развлекательный веб-портал для геймеров, агрегирующий каталог игр, новости игровой индустрии и пользовательские обзоры. Портал должен повышать вовлечённость аудитории за счёт интерактивных функций (лайки, комментарии, рейтинги) и предоставлять администраторам удобный инструмент управления контентом.

### 1.2 Ключевые бизнес-задачи
- **BR-01:** Обеспечить посетителям удобный доступ к структурированной базе компьютерных игр с возможностью фильтрации по жанрам.
- **BR-02:** Предоставить площадку для публикации новостей индустрии с датированием и авторством.
- **BR-03:** Позволить зарегистрированным пользователям оставлять отзывы (обзоры) на игры с числовой оценкой.
- **BR-04:** Внедрить систему ролей (пользователь / администратор) для разграничения прав на создание и модерацию контента.
- **BR-05:** Реализовать личные профили пользователей с историей активности (комментарии, лайки, обзоры).
- **BR-06:** Обеспечить администраторам панель управления с просмотром статистики и возможностью редактирования/удаления контента.
- **BR-07:** Гарантировать корректную работу приложения при пиковых нагрузках до 1000 одновременных пользователей (в рамках учебного/демонстрационного проекта).

### 1.3 Ожидаемые выгоды
- Рост посещаемости портала за счёт актуального контента и удобной навигации.
- Формирование активного сообщества через механизмы обратной связи (отзывы, комментарии).
- Упрощение администрирования контента без привлечения разработчиков.

## 3. Тест-кейсы (примеры для ключевых сценариев)

### TC-01: Регистрация нового пользователя

| Шаг | Действие                                                                 | Ожидаемый результат                                                               |
|-----|--------------------------------------------------------------------------|-----------------------------------------------------------------------------------|
| 1   | Перейти на страницу регистрации                                          | Отображается форма с полями: имя, email, пароль, подтверждение пароля             |
| 2   | Заполнить все поля валидными данными (имя: tester, email: test@mail.com, пароль: 123456) |                                                                                   |
| 3   | Нажать "Зарегистрироваться"                                              | Пользователь создан, происходит автоматический вход, редирект на главную           |
| 4   | Проверить наличие flash-сообщения "Добро пожаловать!"                    | Сообщение отображается                                                             |
| 5   | Выйти из системы, затем войти с email и паролем                          | Вход успешен, пользователь попадает на главную                                     |

### TC-02: Добавление отзыва к игре (пользователь)

| Шаг | Действие                                                                 | Ожидаемый результат                                                               |
|-----|--------------------------------------------------------------------------|-----------------------------------------------------------------------------------|
| 1   | Войти как `gamer@portals.local`                                          | Успешная авторизация                                                               |
| 2   | Перейти на страницу игры "Elden Ring"                                    | Отображается информация об игре, список существующих отзывов, форма для отзыва     |
| 3   | В форме отзыва выбрать оценку 9 из выпадающего списка/звёзд              | Оценка выбрана                                                                     |
| 4   | Ввести текст: "Отличная игра, сложная, но захватывающая"                 |                                                                                   |
| 5   | Нажать "Опубликовать отзыв"                                              | Отзыв появляется в списке отзывов, средний рейтинг игры пересчитывается            |
| 6   | Выйти из системы и зайти под другим пользователем, попытаться редактировать этот отзыв | Кнопки редактирования/удаления отсутствуют (доступно только автору и админу)       |

### TC-03: Фильтрация каталога по жанру (посетитель)

| Шаг | Действие                                                                 | Ожидаемый результат                                                               |
|-----|--------------------------------------------------------------------------|-----------------------------------------------------------------------------------|
| 1   | Перейти на страницу каталога игр                                         | Отображается список всех игр (пагинация, если >10)                                 |
| 2   | В боковой панели фильтров выбрать чекбокс "RPG"                          | Список обновляется (Turbo/Hotwire), отображаются только игры с жанром RPG          |
| 3   | Снять чекбокс и выбрать "FPS"                                            | Отображаются только игры жанра FPS                                                 |
| 4   | Выбрать одновременно "Action" и "Adventure"                              | Показаны игры, у которых есть оба жанра (логика "И")                               |
| 5   | Проверить пагинацию после фильтрации                                     | Количество страниц соответствует отфильтрованному набору                           |

### TC-04: Публикация новости (администратор)

| Шаг | Действие                                                                 | Ожидаемый результат                                                               |
|-----|--------------------------------------------------------------------------|-----------------------------------------------------------------------------------|
| 1   | Войти как `admin@portals.local`                                          | Успешная авторизация, в навигации появляется ссылка "Админ-панель"                 |
| 2   | Перейти в раздел "Новости" → "Создать новость"                           | Отображается форма с полями: заголовок, текст (WYSIWYG/textarea), дата публикации |
| 3   | Заполнить поля и нажать "Опубликовать"                                   | Новость появляется в ленте новостей с указанием автора "admin" и текущей датой    |
| 4   | Выйти и посмотреть ленту новостей как анонимный пользователь             | Новая новость отображается на главной/странице новостей                           |

### TC-05: Административная панель (статистика)

| Шаг | Действие                                                                 | Ожидаемый результат                                                               |
|-----|--------------------------------------------------------------------------|-----------------------------------------------------------------------------------|
| 1   | Войти как администратор                                                  | Успешная авторизация                                                               |
| 2   | Перейти в админ-панель (например, `/admin`)                              | Отображается дашборд с карточками: "Всего игр: 10", "Новостей: 8", "Пользователей: 2", "Отзывов: 5" (значения соответствуют seed-данным) |
| 3   | Нажать на карточку "Игры"                                                | Происходит переход к списку игр с возможностью редактирования/удаления             |


## 4. Требования к проекту (Project Requirements)

### 4.1 Функциональные требования

| ID    | Требование                                                                                                 |
|-------|------------------------------------------------------------------------------------------------------------|
| FR-01 | Система должна предоставлять каталог игр с пагинацией (Kaminari) и фильтрацией по жанрам.                  |
| FR-02 | Карточка игры должна содержать: название, разработчика, жанры (теги), средний рейтинг (от 1 до 10).        |
| FR-03 | Страница игры должна включать список отзывов пользователей и форму добавления отзыва (для авторизованных).  |
| FR-04 | Отзыв должен содержать числовую оценку и текст. Средний рейтинг игры пересчитывается при добавлении/изменении/удалении отзыва. |
| FR-05 | Новости должны иметь заголовок, текст, дату публикации и автора. Публиковать новости может только администратор. |
| FR-06 | Пользователи могут оставлять комментарии к новостям и ставить лайки новостям/отзывам (полиморфная связь).   |
| FR-07 | Аутентификация реализуется через Devise. Минимальная длина пароля — 6 символов.                             |
| FR-08 | Авторизация на основе ролей (пользователь, администратор) с помощью Pundit. Администратор имеет доступ к CRUD-операциям над играми, новостями, отзывами, комментариями. |
| FR-09 | Профиль пользователя отображает имя, email (скрыт), список отзывов, комментариев и лайков.                  |
| FR-10 | Админ-панель содержит сводную статистику по количеству сущностей и ссылки на управление контентом.          |

### 4.2 Нефункциональные требования

| ID    | Требование                                                                                                 |
|-------|------------------------------------------------------------------------------------------------------------|
| NFR-01| Производительность: время отклика сервера на запросы страниц не должно превышать 500 мс при 100 одновременных пользователях. |
| NFR-02| Безопасность: пароли хранятся в зашифрованном виде (bcrypt через Devise). Все действия, изменяющие данные, защищены от CSRF. |
| NFR-03| Доступность: приложение должно быть доступно 24/7 с допуском на плановое обслуживание не более 1 часа в месяц. |
| NFR-04| Масштабируемость: архитектура должна позволять горизонтальное масштабирование веб-серверов и БД.            |
| NFR-05| Юзабилити: интерфейс должен соответствовать принципам адаптивного дизайна (Bootstrap 5), корректно отображаться на мобильных устройствах. |
| NFR-06| Локализация: интерфейс и сообщения должны быть на русском языке (i18n настроен на `ru`).                    |

### 4.3 Технические требования и ограничения

| ID    | Требование                                                                                                 |
|-------|------------------------------------------------------------------------------------------------------------|
| TR-01 | Серверная часть: Ruby 3.2, Rails 7.2.                                                                      |
| TR-02 | База данных: PostgreSQL 16.                                                                                |
| TR-03 | Фронтенд: Bootstrap 5, Hotwire (Turbo + Stimulus) для интерактивных обновлений без перезагрузки страницы.  |
| TR-04 | Аутентификация: Devise. Авторизация: Pundit.                                                               |
| TR-05 | Пагинация: Kaminari.                                                                                       |
| TR-06 | Тестирование: RSpec, FactoryBot, Shoulda Matchers, Capybara. Покрытие кода тестами не менее 80% (по возможности). |
| TR-07 | Развёртывание: поддержка Docker Compose для локального запуска и дальнейшего деплоя.                        |
| TR-08 | Система контроля версий: Git.                                                                              |
| TR-09 | Предоставлены два тестовых аккаунта (admin / user) и seed-данные для демонстрации.                          |

### 4.4 Допущения и ограничения
- Проект учебный/демонстрационный, не предназначен для коммерческого использования в текущем виде.
- На начальном этапе не реализуются загрузка изображений (постеров игр), поиск по тексту (только фильтрация по жанрам), система уведомлений.
- Рейтинг игры вычисляется как среднее арифметическое всех оценок отзывов.
- Роли пользователей предопределены и не могут быть изменены через интерфейс (только вручную через консоль/сиды).
