# Бизнес-требования проекта Portals (BRD)

---

## 1. Введение 

**Portals** — игровой веб-портал, ориентированный на русскоязычную аудиторию геймеров. Платформа агрегирует информацию о компьютерных играх, публикует новости индустрии и предоставляет пользователям возможность делиться отзывами и оценками. Проект реализуется как учебно-демонстрационный с использованием современного стека (Ruby on Rails 7.2, PostgreSQL, Bootstrap 5, Hotwire) и в перспективе может быть расширен до коммерческого продукта.

Основной запрос рынка: создать единое информационное пространство, где игроки могут быстро найти игру по жанру, ознакомиться с оценками сообщества, прочитать актуальные новости и обсудить их — без необходимости посещать несколько разрозненных ресурсов.

## 2. Цели проекта

| Цель | Описание | Измеримый показатель |
|------|----------|----------------------|
| Ц-01 | Привлечение пользователей | Достичь 500 зарегистрированных пользователей в течение 3 месяцев после запуска MVP |
| Ц-02 | Формирование контента | Обеспечить наполнение каталога не менее чем 50 играми и публикацию не менее 20 новостей администратором за первый месяц |
| Ц-03 | Вовлечение аудитории | Среднее количество отзывов на игру ≥ 3, среднее количество комментариев на новость ≥ 2 |
| Ц-04 | Удовлетворённость пользователей | Оценка удобства интерфейса (UI/UX) не ниже 4.0 по 5-балльной шкале на основе опроса первых 100 пользователей |
| Ц-05 | Техническая стабильность | Время отклика сервера ≤ 500 мс при 100 одновременных посетителях; доступность 99,9% в рабочие часы |

## 4. Область применения 

### 4.1 Входит в MVP
- Каталог игр с фильтрацией по жанрам и пагинацией.
- Детальные страницы игр с отзывами и рейтингом.
- Лента новостей с полными текстами, датами и авторами.
- Регистрация и аутентификация (Devise).
- Две роли: пользователь и администратор (Pundit).
- Комментирование новостей, система лайков для новостей и отзывов.
- Личный профиль пользователя с историей активности.
- Административная панель с CRUD для игр, новостей, модерацией отзывов/комментариев и дашбордом статистики.

### 4.2 Не входит в MVP
- Загрузка изображений (постеры игр, аватарки пользователей).
- Полнотекстовый поиск по играм и новостям.
- Система личных сообщений.
- Интеграция с внешними API (Steam, Twitch).
- Мобильное приложение.
- Платёжная система и монетизация.

## 6. Нефункциональные требования 

| Категория | Требование |
|-----------|------------|
| Производительность | Время загрузки любой страницы ≤ 500 мс при 100 одновременных запросах. |
| Доступность | 99,9% uptime в рабочее время (08:00–00:00 МСК), допустимый простой не более 1 часа в месяц. |

| Локализация | Интерфейс на русском языке, формат дат: ДД.ММ.ГГГГ. |
| Поддерживаемые браузеры | Последние версии Chrome, Firefox, Safari, Edge. |

## 7. Ограничения и допущения

- **Бюджет:** проект учебный, трудозатраты ограничены учебным планом (≈ 3–4 человеко-месяца).
- **Технический стек:** жёстко задан (Ruby 3.2, Rails 7.2, PostgreSQL 16, Bootstrap 5, Hotwire, Devise, Pundit, RSpec). Отклонение от стека не допускается.
- **Хостинг:** на начальном этапе разворачивается на локальном сервере или Docker, публичный доступ не обязателен. Для демонстрации используется тестовый домен `ruby-web-portal.aethos.site`.
- **Данные:** первоначальное наполнение осуществляется через `db:seed` (10 игр, несколько новостей, два тестовых аккаунта).
- **Роли:** изменение ролей пользователей через интерфейс не предусмотрено; администратор назначается вручную.
- **Рейтинг игр:** вычисляется как среднее арифметическое всех оценок отзывов и обновляется при каждом добавлении/изменении/удалении.

## 8. Критерии приёмки проекта

Проект считается успешно завершённым, если:
- Все функциональные требования (раздел 5) реализованы и работают без критических ошибок.
- Все нефункциональные требования (раздел 6) подтверждены нагрузочным тестированием и ручной проверкой.
- Набор автоматизированных тестов (RSpec) покрывает ≥ 80% кода и успешно выполняется.
- Предоставлена документация: README с инструкцией по запуску, описание API (если есть), структура БД.
- Проект разворачивается из репозитория одной командой (`docker compose up` или `bin/setup`) и доступен локально.
- Преподавателем проведена демонстрация всех ключевых сценариев согласно пользовательским историям, без неожиданного поведения.

## 9. Пользовательские истории (User Stories)

### 9.1 Анонимный посетитель

**US-01: Просмотр каталога игр с фильтрацией по жанрам**
- **Как:** Анонимный посетитель
- **Я хочу:** На странице «Каталог игр» (`/games`) видеть список всех игр с постраничной навигацией, а также боковую панель с перечнем жанров, чтобы при выборе одного или нескольких жанров список динамически обновлялся, показывая только подходящие игры.
- **Acceptance Criteria:**
  - Страница доступна без авторизации по адресу `/games`.
  - Изначально отображаются все игры (пагинация, если игр > 10).
  - Слева находится панель фильтрации с чекбоксами жанров: Action, Adventure, FPS, Horror, Indie, Puzzle, RPG, Racing, Simulation, Sports, Strategy.
  - При установке флажка «RPG» список мгновенно (через Hotwire/Turbo) обновляется, видны только игры с жанром RPG.
  - При выборе «RPG» и «Action» отображаются игры, у которых указаны оба жанра (логика «И»).
  - Если снять все флажки, возвращается полный список.
  - В каждой карточке игры присутствуют: название, разработчик, жанры, средний рейтинг (число от 1 до 10).

**US-02: Просмотр детальной информации об игре**
- **Как:** Посетитель
- **Я хочу:** Со страницы каталога перейти на страницу отдельной игры (например, `/games/3`), чтобы увидеть её полное описание, разработчика, жанры, средний рейтинг и все оставленные пользователями отзывы.
- **Acceptance Criteria:**
  - Клик по названию игры в каталоге ведёт на `/games/:id`.
  - На странице отображается: название, разработчик, список жанров (теги), средняя оценка (например, «9.0»).
  - Ниже расположен блок «Отзывы» со списком отзывов (автор, дата, текст, оценка).
  - Если отзывов нет, выводится сообщение «Пока нет отзывов».
  - Форма добавления отзыва отображается только для авторизованных пользователей (для гостей скрыта).

**US-03: Чтение новостей**
- **Как:** Посетитель
- **Я хочу:** На странице «Новости» (`/news`) видеть список всех новостей, отсортированных по дате публикации (сначала свежие), с заголовком, кратким текстом и датой.
- **Acceptance Criteria:**
  - Страница доступна по `/news`.
  - Новости отображаются в виде ленты; каждая новость содержит: дату (в формате «ДД.ММ.ГГГГ»), заголовок и первые 2–3 строки текста.
  - Клик по заголовку или кнопке «Читать далее» открывает полную новость по адресу `/news/:id`.
  - На полной странице новости видны: заголовок, полный текст, автор (admin), дата, количество лайков и комментариев.

**US-04: Регистрация нового аккаунта**
- **Как:** Посетитель
- **Я хочу:** На странице регистрации (`/sign_up`) создать учётную запись, заполнив поля: имя пользователя, email, пароль и подтверждение пароля, чтобы затем авторизоваться и пользоваться возможностями портала.
- **Acceptance Criteria:**
  - Ссылка «Регистрация» в шапке сайта ведёт на `/users/sign_up`.
  - Форма содержит обязательные поля: «Имя пользователя», «Email», «Пароль» (минимум 6 символов), «Подтверждение пароля».
  - Валидация на стороне клиента и сервера: email уникален и корректен, пароли совпадают, длина пароля ≥ 6.
  - При успешной отправке создаётся учётная запись, пользователь автоматически входит в систему, перенаправляется на главную страницу с flash-сообщением «Добро пожаловать!».
  - При ошибках форма перерисовывается с соответствующими сообщениями под полями.

**US-05: Вход в систему**
- **Как:** Зарегистрированный пользователь
- **Я хочу:** На странице входа (`/login`) ввести свой email и пароль, чтобы попасть в свой аккаунт и увидеть персональный профиль.
- **Acceptance Criteria:**
  - Ссылка «Войти» в шапке ведёт на `/users/sign_in`.
  - Форма содержит поля «Email» и «Пароль».
  - При корректных данных происходит вход, перенаправление на главную, отображается ссылка на профиль и имя пользователя.
  - При некорректных данных – сообщение «Неверный email или пароль».

### 9.2 Зарегистрированный пользователь

**US-06: Добавление отзыва к игре**
- **Как:** Авторизованный пользователь
- **Я хочу:** На странице конкретной игры (`/games/:id`) заполнить и отправить форму отзыва: выбрать оценку от 1 до 10 (звёзды/слайдер) и написать текст, чтобы мой обзор появился в списке и изменился средний рейтинг игры.
- **Acceptance Criteria:**
  - На странице игры под списком отзывов присутствует форма «Оставить отзыв», если пользователь авторизован.
  - Форма содержит: поле оценки (можно выбрать 1–10, например, выпадающий список или кликабельные звёзды) и текстовое поле (обязательное, минимум 10 символов).
  - После нажатия «Опубликовать» отзыв добавляется на страницу без перезагрузки (Turbo), средний рейтинг игры обновляется.
  - Новый отзыв отображается первым в списке с указанием имени автора и текущей даты.
  - Если поля заполнены некорректно, отображаются ошибки валидации.

**US-07: Просмотр собственного профиля и истории активности**
- **Как:** Пользователь
- **Я хочу:** На странице своего профиля (`/profile`) видеть своё имя, email, а также три вкладки/секции: «Мои отзывы», «Мои комментарии», «Понравившееся», где перечислены все оставленные мной отзывы, комментарии и объекты, которым я поставил лайк.
- **Acceptance Criteria:**
  - В шапке сайта после входа отображается ссылка на профиль (имя пользователя). Клик ведёт на `/users/:id` (или `/profile`).
  - На странице отображается приветствие «Привет, [имя]!» и email.
  - Секция «Мои отзывы» содержит список карточек: название игры, моя оценка, текст отзыва, дата. Есть кнопки «Редактировать» и «Удалить».
  - Секция «Мои комментарии»: список комментариев с текстом, датой и ссылкой на новость.
  - Секция «Понравившееся»: список новостей/отзывов, которые я лайкнул.
  - Данные подгружаются динамически или статически.

**US-08: Комментирование новости**
- **Как:** Пользователь
- **Я хочу:** На странице новости (`/news/:id`) внизу оставить комментарий, введя текст в поле и нажав «Отправить», чтобы он появился под новостью.
- **Acceptance Criteria:**
  - Под полным текстом новости находится форма «Оставить комментарий» с текстовым полем.
  - После отправки комментарий добавляется в список под формой, автор — текущий пользователь, дата публикации.
  - Список комментариев упорядочен по дате (старые сверху или снизу — уточняется дизайном).
  - Без перезагрузки (Turbo).

**US-09: Лайк новости или отзыва**
- **Как:** Пользователь
- **Я хочу:** Рядом с новостью или отзывом видеть кнопку «Нравится» (сердечко), нажать на неё и увидеть увеличение счётчика лайков, выражая одобрение.
- **Acceptance Criteria:**
  - У каждой новости (на `/news/:id`) и у каждого отзыва (на странице игры) присутствует кнопка «❤️ 0» (или иконка).
  - При нажатии кнопка меняет состояние (становится закрашенной), счётчик увеличивается на 1 без перезагрузки.
  - Повторное нажатие убирает лайк, счётчик уменьшается.
  - Действие доступно только авторизованным; гость при клике перенаправляется на страницу входа.

**US-10: Редактирование и удаление собственного отзыва/комментария**
- **Как:** Автор отзыва или комментария
- **Я хочу:** На странице своего отзыва/комментария иметь кнопки «Редактировать» и «Удалить», чтобы изменить текст или полностью удалить запись.
- **Acceptance Criteria:**
  - На странице игры рядом с моим отзывом видны иконки карандаша и корзины.
  - «Редактировать» открывает форму на том же месте (inline) или модальное окно, где можно изменить текст и оценку. После сохранения отзыв обновляется.
  - «Удалить» вызывает подтверждение в диалоговом окне, после подтверждения отзыв удаляется, список отзывов обновляется.
  - Для комментариев — аналогично на странице новости.
  - Пользователь не видит кнопок управления на чужих отзывах.

### 9.3 Администратор

**US-11: Создание новости**
- **Как:** Администратор
- **Я хочу:** В административной панели (`/admin`) перейти в раздел «Новости», нажать «Создать новость», заполнить заголовок, текст и дату, после чего новость опубликуется в общей ленте.
- **Acceptance Criteria:**
  - После входа под admin@portals.local появляется ссылка «Админ-панель» в шапке.
  - В разделе `/admin/news` отображается список новостей с кнопками «Редактировать», «Удалить» и кнопка «Создать новость».
  - Форма создания содержит: «Заголовок» (обязательно), «Текст» (textarea), «Дата публикации» (по умолчанию текущая).
  - После сохранения новость появляется на публичной странице `/news` с автором «admin» и указанной датой.

**US-12: Управление играми (добавление и редактирование)**
- **Как:** Администратор
- **Я хочу:** В админ-панели добавлять новые игры (название, разработчик, жанры, описание) и редактировать существующие, чтобы каталог оставался актуальным.
- **Acceptance Criteria:**
  - В `/admin/games` список игр с кнопками «Редактировать» и «Удалить», а также кнопка «Добавить игру».
  - Форма добавления/редактирования: поля «Название», «Разработчик», «Описание», мультиселект жанров (можно выбрать несколько).
  - При сохранении игра появляется в каталоге, страница обновляется.
  - Администратор не может удалить игру, на которую есть отзывы, без дополнительного подтверждения (или может, с каскадным удалением отзывов — определяется политикой).

**US-13: Модерация отзывов и комментариев**
- **Как:** Администратор
- **Я хочу:** В админ-панели просматривать все отзывы и комментарии, иметь возможность удалить любой нежелательный отзыв или комментарий.
- **Acceptance Criteria:**
  - Раздел `/admin/reviews` показывает таблицу со всеми отзывами (игра, автор, текст, оценка, дата). Рядом — кнопка «Удалить».
  - Аналогично `/admin/comments`.
  - При удалении отзыва пересчитывается средний рейтинг игры.
  - При удалении комментария он исчезает со страницы новости.

**US-14: Просмотр дашборда статистики**
- **Как:** Администратор
- **Я хочу:** На главной странице админ-панели (`/admin`) видеть сводку: общее количество игр, новостей, зарегистрированных пользователей и отзывов, чтобы быстро оценивать активность портала.
- **Acceptance Criteria:**
  - При переходе в `/admin` открывается дашборд с четырьмя карточками-счётчиками: «Игры: N», «Новости: N», «Пользователи: N», «Отзывы: N» (значения динамические).
  - Карточки кликабельны и ведут в соответствующие разделы управления.
  - Данные обновляются при каждом заходе на страницу.
